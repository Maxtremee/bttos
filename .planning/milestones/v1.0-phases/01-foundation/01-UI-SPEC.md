# Phase 1 — UI Design Contract

> Visual and interaction contract for the app foundation: shell, routing, and D-pad spatial navigation.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — plain CSS, hand-rolled components |
| Icon library | none (Phase 1 has no icons; establish in Phase 2 or 3) |
| Font | System default sans-serif — no web font loading; avoids network dependency and latency on first paint. `font-family: system-ui, sans-serif` |

**Rationale (source: STACK.md + objective):** This is a SolidJS + Vite app targeting Chromium 68 on constrained TV hardware. shadcn/Radix are React-based and not applicable. No Tailwind — the build target is a static .ipk bundle; every JS/CSS byte matters. Plain CSS custom properties are used for tokens.

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between tightly grouped elements |
| sm | 8px | Compact element spacing, padding inside small labels |
| md | 16px | Default element padding, gap between list items |
| lg | 24px | Section padding, card inner padding |
| xl | 32px | Layout column gaps, screen-edge padding (minimum safe zone) |
| 2xl | 48px | Major section breaks, vertical rhythm between screen regions |
| 3xl | 64px | Page-level structural spacing (e.g. header height baseline) |

**Exceptions:**
- Focus ring offset: 4px (xs) — consistent with xs, but applied as `outline-offset` not margin
- Touch/remote target minimum: 44px height for any focusable element — D-pad targets must meet this regardless of visual size. This is an override on top of the spacing scale, not a new token.
- Screen safe zone: 48px (2xl) from all four screen edges — accounts for TV overscan. All interactive content must stay within this boundary.

**Source:** Standard 10-foot UI guidelines; overscan safe zone is a TV platform convention.

## Typography

All sizes are in pixels, rendered at 1920×1080 (1080p standard for webOS). Sizes are chosen for legibility from 3 metres (10 feet).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 24px | 400 (regular) | 1.5 | General descriptive text, metadata labels |
| Label | 20px | 600 (semibold) | 1.4 | Button labels, navigation items, card metadata |
| Heading | 32px | 600 (semibold) | 1.2 | Screen titles, section headings |
| Display | 48px | 600 (semibold) | 1.1 | Auth screen device code, prominent single-value displays |

**Weights used:** 400 (regular) and 600 (semibold) only. No other weights.

**Rendering notes:**
- Use `font-smooth: antialiased` — Chromium 68 on webOS benefits from subpixel rendering hints
- Do NOT use `rem` units — the webOS system may override root font size. Use `px` exclusively.
- Minimum legible size at 10 feet on 1080p: 20px. Nothing below 20px in Phase 1.

**Source:** 10-foot UI typography conventions; Netflix, Plex, and Kodi all use 20–48px ranges on 1080p TV displays.

## Color

Dark theme. Standard for TV/media apps. Confirmed in objective context.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0f0f0f` | App background, screen fill |
| Secondary (30%) | `#1a1a1a` | App shell header bar, navigation containers, focused card background |
| Accent (10%) | `#9147ff` | Twitch brand purple — reserved for specific elements listed below |
| Destructive | `#e53935` | Destructive actions only (not in Phase 1; established for downstream phases) |

**Accent reserved for:**
1. Focus ring / focus indicator border on the currently focused D-pad element
2. Active/selected navigation item indicator
3. Primary CTA button fill (when a screen has one)

Accent is NOT used for: body text, icons, backgrounds, non-interactive decorative elements, loading states.

**Focus indicator specification (FNDN-02 critical requirement):**
- `outline: 3px solid #9147ff`
- `outline-offset: 4px`
- Applied via CSS class `.focused` toggled by the spatial navigation library
- Must be visible against both `#0f0f0f` and `#1a1a1a` backgrounds — purple on dark satisfies this
- No `outline: none` anywhere in the codebase. Remove browser default focus styles by replacing them with the custom focus ring, never suppressing focus visibility.

**Text colors:**
- Primary text: `#f0f0f0` (near-white — slightly off-white avoids harsh contrast on dark backgrounds viewed from distance)
- Secondary text: `#a0a0a0` (muted, for metadata, subtitles, helper text)
- Disabled / inactive: `#555555`

**Source:** Twitch brand purple `#9147ff` is the official accent color; dark background values chosen for WCAG AA contrast at 10-foot viewing distance.

## Interaction Contract

This section is specific to Phase 1 because it is establishing the interaction foundation for all subsequent phases.

### D-pad Navigation

| Key | webOS keyCode | Behavior |
|-----|--------------|---------|
| Up | 38 | Move focus to nearest element above (spatial navigation) |
| Down | 40 | Move focus to nearest element below |
| Left | 37 | Move focus to nearest element to the left |
| Right | 39 | Move focus to nearest element to the right |
| OK / Enter | 13 | Activate focused element |
| Back | 461 | Navigate to previous screen; on root screen: show exit confirmation |

**Focus behavior:**
- On screen mount, focus is set to the first focusable element automatically (top-left by default)
- Focus is never lost — if the focused element is removed, focus transfers to the nearest remaining focusable element
- `pointer-events: none` on non-interactive regions to prevent Magic Remote hover from stealing D-pad focus context

### Back Button on Root Screen (CONTEXT.md D-03 — Claude's discretion)

Pattern chosen: **Exit confirmation dialog.** Standard TV app pattern (used by Netflix, YouTube TV, Disney+).

- Pressing Back on the root screen (Channels screen post-login, or Login screen pre-auth) shows a modal overlay: "Exit app?" with two focusable options: "Exit" and "Cancel"
- "Exit" calls `window.close()` (webOS packaged apps respond to this)
- "Cancel" or pressing Back again dismisses the dialog and returns focus to where it was
- Dialog uses the same color tokens: `#1a1a1a` background panel, `#9147ff` accent on the focused option, `#f0f0f0` text

### Screen Transitions

Instant. No CSS transitions, no animation, no fade. Source: CONTEXT.md D-02. The screen swap is a synchronous DOM replacement driven by MemoryRouter route change.

### Focus Indicator Visibility Requirement

The currently focused element must ALWAYS have a visible focus ring. This is a hard requirement from FNDN-02 ("Focus state is visually distinct — user can always see what is selected"). The focus ring must be visible at 10-foot viewing distance — 3px minimum outline width.

## Screen Inventory (Phase 1 Scope)

Phase 1 establishes the shell and routing scaffold. The four screens are skeleton implementations (no real data), used to verify navigation and focus work end-to-end.

| Screen | Route | First Focus | Back Behavior |
|--------|-------|-------------|---------------|
| Login | `/login` | "Sign in" button | Exit confirmation (root) |
| Channels | `/channels` | First channel card (or "Refresh" button if empty) | Exit confirmation (root after auth) |
| Player | `/player/:channel` | Video area (no focusable elements in Phase 1) | Return to Channels |
| Settings | `/settings` | "Log Out" button | Return to Channels |

**Phase 1 scaffolded content:** Each screen shows its name as a heading and one focusable placeholder button. No real API calls. This verifies routing and D-pad navigation work before feature work begins.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (Login screen) | "Sign in with Twitch" |
| Exit confirmation heading | "Exit app?" |
| Exit confirmation confirm | "Exit" |
| Exit confirmation cancel | "Cancel" |
| Empty state heading (Channels) | "No channels live right now" |
| Empty state body (Channels) | "Channels you follow will appear here when they go live. Check back later." |
| Error state (generic) | "Something went wrong. Press OK to try again." |
| Loading state (generic) | "Loading..." |
| Skeleton screen label (Phase 1 placeholder) | "[Screen name] screen — navigation test" |

**Copywriting principles:**
- Short sentences. Read from 10 feet. No jargon.
- "Press OK" not "click" — this is a remote, not a mouse.
- Never expose technical error codes or stack traces to the user.

## CSS Custom Properties Reference

The following CSS custom properties implement the design tokens above. These are established in Phase 1 as the global token foundation for all subsequent phases.

```css
:root {
  /* Colors */
  --color-bg: #0f0f0f;
  --color-surface: #1a1a1a;
  --color-accent: #9147ff;
  --color-destructive: #e53935;
  --color-text-primary: #f0f0f0;
  --color-text-secondary: #a0a0a0;
  --color-text-disabled: #555555;

  /* Spacing */
  --space-xs: 4px;
  --space-sm: 8px;
  --space-md: 16px;
  --space-lg: 24px;
  --space-xl: 32px;
  --space-2xl: 48px;
  --space-3xl: 64px;

  /* Typography */
  --font-size-body: 24px;
  --font-size-label: 20px;
  --font-size-heading: 32px;
  --font-size-display: 48px;
  --font-weight-regular: 400;
  --font-weight-semibold: 600;
  --line-height-body: 1.5;
  --line-height-label: 1.4;
  --line-height-heading: 1.2;
  --line-height-display: 1.1;

  /* Focus */
  --focus-ring: 3px solid var(--color-accent);
  --focus-ring-offset: 4px;

  /* Safe zone */
  --safe-zone: 48px;

  /* Remote target minimum */
  --min-target-height: 44px;
}

/* Global focus rule — never suppress */
:focus-visible {
  outline: var(--focus-ring);
  outline-offset: var(--focus-ring-offset);
}

/* Spatial nav library applies this class */
.focused {
  outline: var(--focus-ring);
  outline-offset: var(--focus-ring-offset);
}
```
