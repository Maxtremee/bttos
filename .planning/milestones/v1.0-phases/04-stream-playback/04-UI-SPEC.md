## Component Inventory

### Video Element

- Full-viewport `<video>` tag, `width: 100%`, `height: 100vh`, `object-fit: cover`
- No native controls (`controls` attribute omitted)
- Background: `--color-bg` while buffering
- Managed by hls.js — do not add `src` attribute directly

### Loading Overlay

- Position: absolute, centered over video area
- Content: single line — "Loading stream..." at body size, `--color-text-secondary`
- Visibility: shown while HLS is acquiring URL and buffering first segment; hidden on first frame
- No spinner image — text-only (no icon library available)

### Info Bar

- Position: absolute, bottom-0, full width, `padding: var(--space-lg) var(--space-xl)`
- Background: `rgba(26, 26, 26, 0.85)` — semi-transparent surface
- Layout: single row — channel name left, game + viewer count right; stream title below channel name as second line
- Auto-hide: visible for 4 seconds after mount and after any remote button press; fades out (CSS transition `opacity 0.3s ease`)
- Show trigger: any remote key event (keydown) while PlayerScreen is mounted
- Contents:
  - Channel name: heading size (32px), semibold, `--color-text-primary`
  - Stream title: body size (24px), regular, `--color-text-secondary`, truncate at 1 line with `text-overflow: ellipsis`
  - Game name: label size (20px), regular, `--color-text-secondary`
  - Viewer count: label size (20px), regular, `--color-text-secondary`, formatted as "123.4K watching"
- No focusable elements in info bar — it is display-only

### Error Overlay

- Position: absolute, centered over video area, full width/height, `background: var(--color-surface)`
- Layout: column, centered, gap `var(--space-md)`
- Contents:
  - Heading (32px, 600): error title (see Copywriting Contract)
  - Body (24px, 400): error detail + instruction (see Copywriting Contract)
  - Retry button: focusable, `background: var(--color-accent)`, `color: var(--color-text-primary)`, `padding: var(--space-md) var(--space-xl)`, `min-height: var(--min-target-height)`, label size (20px, 600)
- Focus: `setFocus('player-retry')` on mount when error overlay renders
- Back key returns to channel list (handled by App.tsx — no changes needed)

### Back Navigation Hint

- Not rendered as a UI component — Back key is already handled globally by App.tsx
- No on-screen "Back" button; TV users understand Back key convention

## Interaction Contract

### Remote Key Mapping (PlayerScreen)

| Key | Action |
|-----|--------|
| Back | Navigate to `/channels` (handled by App.tsx, no changes needed) |
| OK / D-pad any direction | Show info bar (reset 4s auto-hide timer) |
| OK on Retry button | Trigger stream URL re-acquisition + HLS reload |

No play/pause toggle — live streams are always playing. Volume is handled by the TV OS natively.

### State Machine

```
LOADING → PLAYING
LOADING → ERROR
PLAYING → ERROR (stream drops / network loss)
ERROR → LOADING (on Retry)
ERROR → [channels screen] (on Back)
```

### Auto-Retry (network errors)

- On transient HLS network error: exponential backoff, max 3 attempts (delays: 2s, 4s, 8s)
- After 3 failed attempts: transition to ERROR state, show error overlay with Retry button
- On fatal HLS error (stream offline): transition immediately to ERROR state

### Focus Initialization

- On LOADING state mount: no focusable element — info bar hides after 4s, no focus needed
- On ERROR state mount: `setFocus('player-retry')` immediately
- On PLAYING state: no persistent focus — remote events trigger info bar reveal
