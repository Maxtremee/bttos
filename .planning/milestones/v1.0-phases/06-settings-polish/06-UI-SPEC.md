## Spacing Scale

Declared values — all tokens already defined in `src/styles/global.css` `:root`. Do not redefine; reference these variables directly.

| Token | CSS Variable | Value | Usage |
|-------|-------------|-------|-------|
| xs | `--space-xs` | 4px | Icon gaps, inline padding |
| sm | `--space-sm` | 8px | Compact element spacing |
| md | `--space-md` | 16px | Default element spacing, button padding |
| lg | `--space-lg` | 24px | Section padding, info bar padding |
| xl | `--space-xl` | 32px | Layout gaps, button horizontal padding |
| 2xl | `--space-2xl` | 48px | Screen-level padding (all existing screens use this) |
| 3xl | `--space-3xl` | 64px | Major section breaks |

Exceptions:
- Safe zone: `--safe-zone: 48px` — apply as minimum screen-edge padding on all sides of the settings screen. All existing screens already use `--space-2xl` (48px) as top/left padding, which satisfies this constraint.
- Remote target minimum: `--min-target-height: 44px` — every focusable button in settings must set `min-height: var(--min-target-height)`.
- Focus ring offset: 4px (defined as `--focus-ring-offset`) — all Focusable elements inherit this via `.focused` class.
- Player settings overlay: internal padding is `--space-lg` (24px) on all sides; do not use `--space-2xl` as the overlay is not full-screen.

Source: `src/styles/global.css`.

## Color

All color tokens already defined in `src/styles/global.css`. Do not introduce new color values; reference existing variables only.

| Role | CSS Variable | Hex | Usage |
|------|-------------|-----|-------|
| Dominant (60%) | `--color-bg` | `#0f0f0f` | Full-screen background, player settings overlay backdrop base |
| Secondary (30%) | `--color-surface` | `#1a1a1a` | Settings screen cards/rows, confirmation dialog background, player overlay panel background |
| Accent (10%) | `--color-accent` | `#9147ff` | Reserved — see list below |
| Destructive | `--color-destructive` | `#e53935` | Log Out button background only |
| Text primary | `--color-text-primary` | `#f0f0f0` | All headings, button labels, toggle on-state label |
| Text secondary | `--color-text-secondary` | `#a0a0a0` | Preference descriptions, toggle off-state label, overlay status text |
| Text disabled | `--color-text-disabled` | `#555555` | Inactive toggle track, hint text that auto-hides |

Accent (`#9147ff`) reserved exclusively for:
- Focus ring on all focusable elements (via `--focus-ring: 3px solid var(--color-accent)`)
- Gear icon foreground color when focused
- Confirmation dialog "cancel" button background (non-destructive primary action)

Accent is NOT used for:
- Log Out button (use `--color-destructive`)
- Preference row backgrounds
- Toggle track when off

Player settings overlay background: `rgba(26, 26, 26, 0.92)` — matches existing info bar pattern (`rgba(26, 26, 26, 0.85)`) but slightly more opaque since it requires active D-pad interaction.

Source: `src/styles/global.css`, `src/screens/PlayerScreen.tsx` info bar pattern.

## Component Inventory

New components required for Phase 6. Each maps directly to a CONTEXT.md decision.

### SettingsScreen (replacement)
- Full-screen layout at `/settings` route
- `<main>` with `padding: var(--space-2xl)` (matches all other screens)
- `<h1>` at `--font-size-heading` / `--font-weight-semibold`: "Settings"
- Two preference rows: "Chat visibility" (toggle) and "Chat position" (toggle)
- One destructive action row: "Log Out" button
- D-pad focus order: gear icon (header, from ChannelsScreen) → chat visibility toggle → chat position toggle → Log Out button

### GearIconButton
- Placed in ChannelsScreen header, top-right area
- Minimum size: 44px × 44px (`--min-target-height`)
- Icon: Unicode `⚙` (U+2699) at `--font-size-heading` (32px), or inline SVG gear at 32px
- Color: `--color-text-secondary` at rest; `--color-text-primary` when `.focused`
- Focus ring: standard `var(--focus-ring)` / `var(--focus-ring-offset)` via `.focused` class
- D-pad: reachable by pressing Up from the channel grid

### ToggleControl
- Displays a label + on/off indicator inline
- Label at `--font-size-body` / weight 400 / `--color-text-primary`
- State indicator text: "On" / "Off" at `--font-size-label` / weight 600
  - On: `--color-accent`
  - Off: `--color-text-disabled`
- Min height: `var(--min-target-height)` (44px)
- OK/Enter press: toggles value and writes to localStorage immediately
- Focus ring: standard `.focused` class

### LogoutButton
- Background: `--color-destructive` (`#e53935`)
- Label: "Log Out" at `--font-size-label` / `--font-weight-semibold` / `--color-text-primary`
- Min height: `var(--min-target-height)`
- Padding: `var(--space-md)` vertical × `var(--space-xl)` horizontal (matches existing button pattern)
- OK/Enter press: opens LogoutConfirmDialog — does NOT immediately log out

### LogoutConfirmDialog
- Modal overlay: `position: fixed`, full viewport, z-index: 200
- Backdrop: `rgba(0, 0, 0, 0.7)`
- Dialog panel: `background: var(--color-surface)`, centered, `padding: var(--space-xl)`
- Heading: "Log out of Twitch?" at `--font-size-heading` / `--font-weight-semibold`
- Body: "You will need to sign in again on your phone or computer." at `--font-size-body` / `--color-text-secondary`
- Two buttons side by side: "Cancel" (accent background) and "Log Out" (destructive background)
- Default focus on Cancel — prevents accidental logout (D-01 → D-09 decision)
- Pressing Back key also dismisses (Cancel behavior) — consistent with App.tsx Back key pattern

### PlayerSettingsOverlay
- Triggered by Green button (keyCode TBD — App.tsx global handler, CONTEXT.md D-04)
- Overlay renders on top of the playing video; stream continues playing underneath
- Position: right side of the video area (matches chat sidebar being on the right by default)
- Width: 320px fixed; height: auto
- Background: `rgba(26, 26, 26, 0.92)`
- Internal padding: `var(--space-lg)` all sides
- Section title: "Chat Settings" at `--font-size-heading` / `--font-weight-semibold`
- Contains exactly 2 preference controls: chat visibility toggle + chat position toggle
- Does NOT contain Log Out — full settings only from ChannelsScreen (CONTEXT.md D-08)
- Dismiss: Green button again, or Back key
- Focus: on open, first toggle receives focus; on close, focus returns to video area

## Interaction Contracts

### Remote Key Assignments (Phase 6 additions)

| Key | keyCode | Action | Screen |
|-----|---------|--------|--------|
| Green | 404 | Open PlayerSettingsOverlay | PlayerScreen |
| Green | 404 | Navigate to `/settings` | ChannelsScreen |
| Back (461) | 461 | Dismiss dialog/overlay (Cancel) | LogoutConfirmDialog, PlayerSettingsOverlay |

Note: keyCode 404 is the standard webOS Green button. Verify against real hardware. Add to App.tsx global handler alongside existing Back (461) handler.

### Preference Persistence

- Key: `twitch_prefs` in localStorage
- Format: JSON object `{ chatVisible: boolean, chatPosition: 'left' | 'right' }`
- Default values (first launch, key absent): `{ chatVisible: true, chatPosition: 'right' }`
- Write: on every toggle change, immediately — no "Save" button required
- Read: on PlayerScreen mount, before chat sidebar renders

### Chat Position Integration

- `chatPosition: 'right'` → ChatSidebar rendered after video `<div>` in flex row (current behavior)
- `chatPosition: 'left'` → ChatSidebar rendered before video `<div>` in flex row
- `chatVisible: false` default → `chatVisible` signal in PlayerScreen initializes from prefs; Red button still toggles live

### D-pad Navigation — Full App Audit Requirements (Success Criteria #3)

The executor must verify no dead ends exist across all screens after Phase 6:

| Screen | Up | Down | Left | Right | OK | Back |
|--------|----|----|------|-------|----|------|
| LoginScreen | — | — | — | — | QR hint | exit app |
| ChannelsScreen | gear icon | channel grid | in grid | in grid | open player | exit app |
| ChannelsScreen (gear focused) | — | channel grid | — | — | go to /settings | — |
| PlayerScreen | — | — | — | — | — | go to /channels |
| PlayerScreen (error overlay) | — | — | — | — | retry | go to /channels |
| SettingsScreen | log out | toggle 1 | — | — | activate | go to /channels |
| LogoutConfirmDialog | — | — | Cancel | Log Out | activate focused | dismiss (Cancel) |
| PlayerSettingsOverlay | toggle 2 | toggle 1 | — | — | activate | dismiss |
