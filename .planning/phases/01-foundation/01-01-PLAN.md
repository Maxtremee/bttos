After scaffold, rewrite `vite.config.ts` to exactly:

```typescript
import { defineConfig } from 'vite'
import solid from 'vite-plugin-solid'

export default defineConfig({
  plugins: [solid()],
  build: {
    target: ['chrome68'],
    outDir: 'dist',
    assetsInlineLimit: 0,
  },
  base: './',
})
```

Key constraints (per RESEARCH.md Pattern 1 and Anti-Patterns):
- `base: './'` is MANDATORY — webOS apps run from file:// origin; absolute paths (base: '/') will 404
- `build.target: ['chrome68']` is MANDATORY — webOS 5.x baseline; do NOT omit this
- `assetsInlineLimit: 0` — avoid base64 inlining for file:// origin compatibility
- Do NOT use `@vitejs/plugin-legacy` — `build.target` handles transpilation correctly
- Do NOT set `base: '/'` (default Vite) — this will break the app on real hardware

Keep the default `tsconfig.json` from the solid-ts template unchanged (it already configures SolidJS JSX transform). If the template produces `tsconfig.node.json`, keep it.

Remove the default Vite boilerplate: `src/App.css`, `src/logo.svg`, `src/assets/react.svg` (if present). Keep `src/App.tsx` and `src/index.tsx` (or `src/main.tsx`) as stubs — they will be replaced in Plan 03.

Verify the build works:
```bash
npm run build
```

Must exit 0. Must produce `dist/index.html`. Check that asset paths in `dist/index.html` are relative (start with `./assets/` not `/assets/`).
