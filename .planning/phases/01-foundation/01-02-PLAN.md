<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| appinfo.json → webOS platform | Manifest is read by ares-package and webOS launcher; id and version values affect app identity |
| navigation module → D-pad input | Spatial nav library processes keydown events for directional focus; input comes from hardware remote |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | appinfo.json id field | accept | App id is source-controlled; only changed by developer; no runtime mutation possible |
| T-02-02 | Elevation of Privilege | window.close() in exit dialog | mitigate | window.close() is only reachable via explicit D-pad select on the "Exit" button in ExitConfirmDialog; not called from navigation module; plan 03 implements the guard |
| T-02-03 | Denial of Service | @lampa-dev/solidjs-spatial-navigation init failure | mitigate | initSpatialNav() is called before render in main.tsx; if init() throws, app will fail with a visible console error rather than silently degrading; fallback documented in RESEARCH.md Pitfall 6 |
</threat_model>
