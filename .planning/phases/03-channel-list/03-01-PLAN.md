<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Twitch Helix API -> App | API response data (stream titles, game names) rendered in DOM |
| Auth token -> Helix API | Bearer token sent in Authorization header over HTTPS |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | Stream title / game name in DOM | mitigate | SolidJS JSX auto-escapes all string interpolation — no innerHTML or dangerouslySetInnerHTML used. Stream titles with `<script>` tags render as text. |
| T-03-02 | Information Disclosure | Auth token in TwitchChannelService | mitigate | Token read from authStore (reactive store), never logged to console. Headers object is ephemeral (created per request). No `console.log(token)` in service code. |
| T-03-03 | Information Disclosure | CLIENT_ID in source | accept | Client ID is a public identifier for Twitch apps (not a secret). It is embedded in the built JS bundle as expected for public OAuth clients. |
| T-03-04 | Denial of Service | Unbounded pagination loop | mitigate | Pagination loop terminates when `cursor` is absent. Add a safety cap of 50 pages (5000 channels) to prevent infinite loops from malformed API responses. |
</threat_model>
