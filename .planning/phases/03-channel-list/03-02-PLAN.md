<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Twitch API data -> ChannelGrid DOM | Stream titles and game names rendered via SolidJS JSX (auto-escaped) |
| User input (D-pad OK) -> Router navigation | user_login from API data used in navigation path |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Tampering | user_login in navigation path | mitigate | `user_login` is used in `navigate('/player/' + user_login)` — the router only matches `/player/:channel` which renders PlayerScreen. No injection possible since SolidJS router treats the segment as a single param. |
| T-03-06 | Denial of Service | setInterval polling every 60s | accept | 60s interval is conservative. Each poll makes 1 + ceil(N/100) API calls where N is followed count. For 1000 follows, that is 11 calls per minute — well within Twitch rate limits (800 points/min). |
| T-03-07 | Spoofing | Stale data rendered after token expiry | mitigate | Token refresh is handled in TwitchChannelService (Plan 01). If refresh fails, authStore clears and AuthGuard redirects to login. ChannelsScreen shows error state if fetch fails. |
</threat_model>
