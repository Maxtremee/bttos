## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, viewer count icon gap |
| sm | 8px | Card internal element gaps (title-to-meta gap) |
| md | 16px | Card internal padding, metadata row spacing |
| lg | 24px | Grid column/row gap |
| xl | 32px | Grid outer padding (horizontal) |
| 2xl | 48px | Screen safe-zone padding (matches `--safe-zone`) |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Safe zone inset is 48px (`--safe-zone`) applied to the screen's outer padding — aligns with webOS overscan area.
- Channel card thumbnail height: 160px (16:9 at a grid-friendly size). Not on the 8-pt scale but a fixed media dimension.
- Focus ring offset: 4px (`--focus-ring-offset`).

## Component Inventory

### ChannelCard

A focusable card in the channel grid. Contains:

1. **Thumbnail** — `<img>` using Twitch thumbnail URL template `{width}x{height}` resolved to `284x160` (16:9). Fallback: `--color-surface` placeholder div when image fails to load. `loading="lazy"` attribute set.
2. **Card body** — `padding: var(--space-md)`, `background: var(--color-surface)`.
3. **Stream title** — font-size `--font-size-body`, weight `--font-weight-regular`, color `--color-text-primary`. Truncated to 2 lines (`-webkit-line-clamp: 2`).
4. **Game name** — font-size `--font-size-label`, weight `--font-weight-regular`, color `--color-text-secondary`. Single line, truncated.
5. **Viewer count** — font-size `--font-size-label`, weight `--font-weight-regular`, color `--color-text-secondary`. Format: "{N} viewers" (abbreviated: "1.2K viewers" above 1000).

Focus state: `.focused` class applied by `@lampa-dev/solidjs-spatial-navigation`. Applies `outline: var(--focus-ring)` (3px solid `#9147ff`) with `outline-offset: var(--focus-ring-offset)` (4px). The card surface does NOT change color on focus — only the ring is applied.

Card minimum height: auto (content-driven). Card minimum touch/remote target: the full card is the focusable element — no minimum height constraint needed since cards are inherently larger.

### ChannelGrid (layout)

CSS Grid layout. Columns: `repeat(4, 1fr)` at 1920px wide (standard TV resolution). Gap: `var(--space-lg)` (24px) between cards. Outer padding: `var(--space-2xl)` (48px) matching `--safe-zone`.

Scroll: If more than one row of cards, the grid scrolls vertically. `overflow-y: auto`, `scroll-behavior: smooth`. Focus management via `@lampa-dev/solidjs-spatial-navigation` handles scroll-into-view automatically.

`FocusableGroup` wraps the entire grid with `focusKey="channels-grid"`. Each card uses `Focusable` with `focusKey="channel-{user_login}"`.

### LoadingState

Full-screen centered layout. Single line of text using `--font-size-body`, `--font-weight-regular`, `--color-text-secondary`. Text: "Loading channels..." No spinner — text only (avoids animation cost on low-end hardware).

### EmptyState

Full-screen centered layout. Two lines:
- Heading: "No channels live right now" — `--font-size-heading`, `--font-weight-semibold`, `--color-text-primary`.
- Body: "Check back later or follow more channels on Twitch" — `--font-size-body`, `--font-weight-regular`, `--color-text-secondary`.

No actions on the empty state. The Back key remains functional to exit the app (handled by global handler in App.tsx).

### ErrorState

Full-screen centered layout. Two lines:
- Heading: "Could not load channels" — `--font-size-heading`, `--font-weight-semibold`, `--color-text-primary`.
- Body: "Check your connection and press OK to retry" — `--font-size-body`, `--font-weight-regular`, `--color-text-secondary`.

A single focusable retry button below the body text. Button style matches the LoginScreen retry button: `background: var(--color-accent)`, `color: var(--color-text-primary)`, `padding: var(--space-md) var(--space-xl)`, `font-size: var(--font-size-label)`, `font-weight: var(--font-weight-semibold)`, `border: none`. Focus: `.focused` class. Label: "Retry".

## Interaction Contract

### D-pad Navigation

- On screen mount: `setFocus` targets the first channel card in the grid (`channel-{first_user_login}`). If grid is empty or loading, focus is not set (no focusable element).
- Left/right: moves focus between cards in the same row.
- Up/down: moves focus between rows. `@lampa-dev/solidjs-spatial-navigation` handles this automatically via `FocusableGroup`.
- OK on a card: navigates to `/player/{user_login}` via `useNavigate`.
- Back key: handled globally in App.tsx — opens ExitConfirmDialog if on channels screen (or navigates back in stack if deeper).

### Auto-Refresh

Polling interval: 60 seconds. Implemented via `setInterval` in `onMount`, cleared in `onCleanup`.

On each poll: re-fetch followed channels + live streams. Diff the result against current list:
- New live channels appear appended (or re-sorted) in the grid.
- Channels that went offline disappear from the grid.
- Focus position is preserved — do not reset focus on refresh. If the focused card disappears, focus falls back to the first card in the updated grid.

No visible refresh indicator. The update is silent.

### OK Action on Channel Card

On OK press, navigate immediately to `/player/{user_login}`. No confirmation. Phase 4 handles playback initiation.

### No Destructive Actions

This phase contains no destructive actions (no delete, no logout). The destructive color (`--color-destructive`) is not used.
