**PlayerScreen.tsx structure:**

1. **Imports:** `solid-js` (createSignal, createResource, createEffect, onMount, onCleanup, Show), `@solidjs/router` (useParams), `hls.js` (Hls), navigation (Focusable, useSpatialNavigation), services (twitchStreamService, authStore), types (StreamData).

2. **State signals:**
   - `const [playerState, setPlayerState] = createSignal<'loading' | 'playing' | 'error'>('loading')`
   - `const [errorKind, setErrorKind] = createSignal<'offline' | 'network' | 'unknown'>('unknown')`
   - `const [infoVisible, setInfoVisible] = createSignal(true)`
   - `let videoRef!: HTMLVideoElement`
   - `let hls: Hls | undefined`
   - `let hideTimer: ReturnType<typeof setTimeout> | undefined`
   - `let retryCount = 0` (for exponential backoff, max 3)

3. **Stream data fetch:** Use `createResource` with `params.channel` to fetch StreamData from `https://api.twitch.tv/helix/streams?user_login=${login}` using `authStore.token` and `import.meta.env.VITE_TWITCH_CLIENT_ID` for headers. Return `data.data[0] as StreamData | null`.

4. **formatWatching helper:** Format viewer count — `count >= 1000 ? \`${(count / 1000).toFixed(1)}K watching\` : \`${count} watching\``. Note: uses "watching" not "viewers" per UI-SPEC copywriting contract.

5. **initPlayer async function:**
   - Set `playerState('loading')`, `retryCount = 0`
   - Try: `const m3u8Url = await twitchStreamService.fetchStreamM3u8Url(params.channel)`
   - If `!Hls.isSupported()`: set error state, errorKind 'unknown', return
   - Create `hls = new Hls({ maxBufferLength: 30, maxBufferSize: 30_000_000, backBufferLength: 0, liveSyncDurationCount: 3 })`
   - `hls.attachMedia(videoRef)` then `hls.loadSource(m3u8Url)`
   - On `Hls.Events.MANIFEST_PARSED`: `videoRef.play().catch(() => {})`, `setPlayerState('playing')`
   - On `Hls.Events.ERROR`: if `!data.fatal` return. If `data.type === Hls.ErrorTypes.NETWORK_ERROR` and `retryCount < 3`: increment retryCount, setTimeout with `2000 * Math.pow(2, retryCount - 1)` delay calling `hls.startLoad()`. If `data.type === Hls.ErrorTypes.MEDIA_ERROR`: `hls.recoverMediaError()`. Otherwise: `hls.destroy()`, determine errorKind ('network' for NETWORK_ERROR, 'offline' if error detail includes 'manifestLoadError' after retries, 'unknown' otherwise), `setPlayerState('error')`
   - Catch: `setErrorKind('network')`, `setPlayerState('error')`

6. **handleRetry function:** Destroy existing hls (`hls?.destroy(); hls = undefined`), call `initPlayer()`

7. **Info bar auto-hide logic:**
   - `showInfoBar()`: `setInfoVisible(true)`, `clearTimeout(hideTimer)`, `hideTimer = setTimeout(() => setInfoVisible(false), 4000)`
   - On mount: call `showInfoBar()`, add `window.addEventListener('keydown', showInfoBar)`
   - On cleanup: `window.removeEventListener('keydown', showInfoBar)`, `clearTimeout(hideTimer)`, `hls?.destroy()`

8. **onMount:** Call `initPlayer()`, call `showInfoBar()`
9. **onCleanup:** Remove keydown listener, clear hideTimer, destroy hls

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Route param -> service call | Channel login from `/player/:channel` passed to TwitchStreamService |
| App -> Twitch Helix | OAuth Bearer token sent to fetch stream metadata for info bar |
| HLS.js -> Usher CDN | HLS segments fetched from Twitch CDN via URLs from Usher |
| Remote keydown -> UI state | Any keydown event triggers info bar visibility toggle |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | Tampering | PlayerScreen route param | mitigate | channelLogin validated by TwitchStreamService.validateChannelLogin before any fetch call |
| T-04-05 | Denial of Service | PlayerScreen error retry loop | mitigate | Exponential backoff (2s, 4s, 8s) with max 3 auto-retries; after 3 failures, require manual Retry press |
| T-04-06 | Information Disclosure | Helix stream metadata fetch | accept | Uses standard Bearer token to public Helix endpoint; only returns public stream info |
| T-04-07 | Denial of Service | hls.js buffer exhaustion | mitigate | maxBufferLength: 30s, maxBufferSize: 30MB, backBufferLength: 0 — prevents RAM exhaustion on TV hardware |
</threat_model>
