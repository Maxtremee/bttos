Create `src/services/EmoteService.ts`:

```typescript
export type EmoteMap = Map<string, string>  // emote code -> image URL

export class EmoteService {
  private cache = new Map<string, EmoteMap>()

  async getEmoteMap(broadcasterId: string): Promise<EmoteMap> {
    const cached = this.cache.get(broadcasterId)
    if (cached) return cached
    const map = await this.buildEmoteMap(broadcasterId)
    this.cache.set(broadcasterId, map)
    return map
  }

  async buildEmoteMap(broadcasterId: string): Promise<EmoteMap> {
    const map: EmoteMap = new Map()
    // Fetch all providers in parallel, each wrapped in try/catch
    await Promise.all([
      this.fetchBttvGlobal(map),
      this.fetchBttvChannel(map, broadcasterId),
      this.fetchSevenTvGlobal(map),
      this.fetchSevenTvChannel(map, broadcasterId),
      this.fetchFfzGlobal(map),
      this.fetchFfzChannel(map, broadcasterId),
    ])
    return map
  }
  // ... private methods for each provider
}
```

Private methods — exact fetch URLs and response parsing:

- `fetchBttvGlobal(map)`: GET `https://api.betterttv.net/3/cached/emotes/global`. Response: `BttvEmote[]` where `BttvEmote = { id: string; code: string }`. For each: `map.set(e.code, \`https://cdn.betterttv.net/emote/${e.id}/2x\`)`.

- `fetchBttvChannel(map, broadcasterId)`: GET `https://api.betterttv.net/3/cached/users/twitch/${broadcasterId}`. Response: `{ channelEmotes: BttvEmote[], sharedEmotes: BttvEmote[] }`. Iterate both arrays, same CDN pattern.

- `fetchSevenTvGlobal(map)`: GET `https://7tv.io/v3/emote-sets/global`. Response: `{ emotes: [{ name: string, data: { host: { url: string } } }] }`. For each: `map.set(e.name, \`https:${e.data.host.url}/2x.webp\`)`. Always use `.webp` — never AVIF (Chromium 68 compatibility per Research Pitfall 4).

- `fetchSevenTvChannel(map, broadcasterId)`: GET `https://7tv.io/v3/users/twitch/${broadcasterId}`. Response: `{ emote_set: { emotes: [...] } }`. Same CDN pattern as global.

- `fetchFfzGlobal(map)`: GET `https://api.frankerfacez.com/v1/set/global`. Response: `{ default_sets: number[], sets: { [setId]: { emoticons: [{ id: number, name: string }] } } }`. For each set in default_sets, iterate emoticons: `map.set(e.name, \`https://cdn.frankerfacez.com/emote/${e.id}/2\`)`.

- `fetchFfzChannel(map, broadcasterId)`: GET `https://api.frankerfacez.com/v1/room/id/${broadcasterId}`. Response: `{ sets: { [setId]: { emoticons: [...] } } }`. Same CDN pattern.

Each method wrapped in try/catch — on failure, log a warning and return (do not throw, do not break other providers).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| App -> EventSub WebSocket | Outbound WSS to Twitch; messages are server-pushed JSON |
| App -> Helix API | Outbound HTTPS POST for subscription; bearer token in header |
| App -> Third-party emote APIs | Outbound HTTPS GET to BTTV, 7TV, FFZ; no auth required |
| EventSub -> App | Inbound WebSocket messages; must validate message_type before processing |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Spoofing | TwitchChatService | accept | WebSocket connects to hardcoded wss://eventsub.wss.twitch.tv/ws (TLS); no user-controlled URL. Reconnect URL from session_reconnect is also Twitch-origin. |
| T-05-02 | Tampering | EventSub message parsing | mitigate | Validate message_type field before processing; do not blindly trust payload structure. Use optional chaining for nested fields (emote?.id, message?.fragments). |
| T-05-03 | Information Disclosure | Bearer token in subscribe POST | mitigate | Token sent only to api.twitch.tv over HTTPS. CLIENT_ID from env var, never hardcoded. |
| T-05-04 | Denial of Service | High-traffic chat flooding DOM | mitigate | Cap message store at 150 items (splice from front). Batch incoming messages every 100ms. |
| T-05-05 | Elevation of Privilege | Scope error on subscribe | mitigate | On 403 from subscribe, call onScopeError callback which triggers re-auth flow. Do not retry with same token. |
| T-05-06 | Tampering | Third-party emote API responses | accept | Emote URLs are CDN image URLs rendered as <img> src — no script injection possible. Alt text is emote code string. Low risk. |
</threat_model>
