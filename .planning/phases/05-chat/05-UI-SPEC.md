## Spacing Scale

Declared values (must be multiples of 4). All tokens inherited from `src/styles/global.css`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gap between emote image and adjacent text within a message fragment |
| sm | 8px | Vertical padding between individual chat message rows |
| md | 16px | Horizontal padding inside the chat sidebar panel |
| lg | 24px | Vertical padding at top of chat sidebar panel |
| xl | 32px | Not used in chat sidebar directly |
| 2xl | 48px | Not used in chat sidebar |
| 3xl | 64px | Not used in chat sidebar |

Exceptions:
- Chat sidebar panel width: 360px fixed (TV 10-foot rule — chat should not dominate the video). This is a layout constant, not a spacing token.
- Emote image size: 28px × 28px (vertical-align: middle). Chosen to match --font-size-body (24px) with 2px vertical breathing room. Not a spacing token.
- Remote touch target minimum: 44px (--min-target-height) — applies to the toggle affordance hint if rendered as a visible button.

Source: spacing scale from `src/styles/global.css` (RESEARCH.md confirms no new dependencies; layout constants are phase-specific decisions).

## Typography

All sizes and weights inherit from `src/styles/global.css`. No new typography tokens.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 24px (--font-size-body) | 400 (--font-weight-regular) | 1.5 (--line-height-body) |
| Label | 20px (--font-size-label) | 400 (--font-weight-regular) | 1.4 (--line-height-label) |
| Heading | 32px (--font-size-heading) | 600 (--font-weight-semibold) | 1.2 (--line-height-heading) |
| Display | 48px (--font-size-display) | 600 (--font-weight-semibold) | 1.1 (--line-height-display) |

Chat-specific typography assignments:
- Username in chat message: 20px (--font-size-label), weight 600 (--font-weight-semibold), color = chatter's color from EventSub (or --color-text-secondary if absent)
- Message body text: 20px (--font-size-label), weight 400 (--font-weight-regular), color --color-text-primary
- Chat loading/error notice: 20px (--font-size-label), weight 400, color --color-text-secondary
- Overlay toggle hint: 20px (--font-size-label), weight 400, color --color-text-disabled

Rationale: using --font-size-label (20px) rather than --font-size-body (24px) for chat messages. Chat is a secondary information stream over video; using the body size would visually compete with the stream. 20px is still readable at 10-foot TV viewing distance.

Source: `src/styles/global.css` (all tokens pre-existing).

## Color

All values inherited from `src/styles/global.css`. No new color tokens.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #0f0f0f (--color-bg) | Video area background |
| Secondary (30%) | #1a1a1a (--color-surface) | Chat sidebar panel background (opaque — sidebar is a separate region, not an overlay) |
| Accent (10%) | #9147ff (--color-accent) | Focus ring on the toggle hint affordance only |
| Destructive | #e53935 (--color-destructive) | Not used in Phase 5 — no destructive actions |

Accent reserved for:
1. Focus ring (--focus-ring: 3px solid var(--color-accent)) on the chat toggle hint when it receives spatial navigation focus.
2. No other use in Phase 5. Chatter username colors come from EventSub event data, not the design system accent.

Chat sidebar background: `#1a1a1a` (--color-surface), fully opaque. The sidebar is a distinct layout region beside the video, not a floating overlay — transparency would look wrong against the hard video edge.

Text colors in chat:
- Username: chatter's color from EventSub `event.color` field. If absent or `#000000`, fall back to --color-text-secondary (#a0a0a0).
- Message body: --color-text-primary (#f0f0f0)
- Timestamps / meta text: --color-text-secondary (#a0a0a0)
- Error / status text: --color-text-secondary (#a0a0a0)

Source: `src/styles/global.css` (all tokens pre-existing). Pattern from PlayerScreen.tsx info bar.

## Layout Contract

### PlayerScreen Layout (with chat)

- PlayerScreen uses a horizontal flex container: `display: flex; width: 100vw; height: 100vh`.
- When chat is visible: video area takes `flex: 1` (fills remaining width), chat sidebar is `width: 360px; flex-shrink: 0` on the right.
- When chat is hidden: video area takes full `100vw` width — sidebar unmounted.
- The video `<video>` element fills its container and maintains aspect ratio via `object-fit: contain` (letterboxed if needed). This means the stream shrinks horizontally when chat is open, not cropped.
- Info bar overlays the video area only (not the sidebar). Its positioning and z-index remain unchanged from Phase 4.

### Chat Sidebar Panel

- Position: static flex child, rightmost in the PlayerScreen flex container. Not absolutely positioned.
- Dimensions: width 360px fixed, height 100vh.
- Background: #1a1a1a (--color-surface), fully opaque.
- Border: 1px solid rgba(255,255,255,0.1) on the left edge only — subtle separator between video and chat.
- Overflow: `overflow: hidden` on the container (no visible scrollbar — chat is passive display).
- Scroll: CSS `flex-direction: column-reverse` for native auto-scroll to newest messages. Messages array rendered newest-first inside the flex container. Chromium 68 fallback: imperatively set `scrollTop = scrollHeight` inside `requestAnimationFrame` if column-reverse exhibits rendering bugs on device (RESEARCH.md Pitfall 5).
- Padding: 24px (--space-lg) top, 16px (--space-md) left/right, 0 bottom (messages scroll to edge).

### Message List

- Max messages in DOM: 150 (splice from front when exceeded — RESEARCH.md Pattern 4).
- Batching: buffer incoming messages for 100ms before appending to store (RESEARCH.md Pattern 6) — prevents layout thrashing in high-traffic chat.
- Each message row: display flex, align-items flex-start, gap --space-xs (4px), padding-block --space-sm (8px).
- Username + colon: inline, color from EventSub or fallback. No line break between username and message body.
- Message body: fragments rendered inline — text spans and emote `<img>` elements in a single wrapping inline flow.

### Emote Images

- Native Twitch emotes: `<img src="https://static-cdn.jtvnw.net/emoticons/v2/{id}/static/dark/2.0" width="28" height="28" alt="{text}" style="vertical-align: middle; display: inline-block">`
- BTTV emotes: `<img src="https://cdn.betterttv.net/emote/{id}/2x" width="28" height="28" alt="{code}" style="vertical-align: middle; display: inline-block">`
- 7TV emotes: `<img src="https://cdn.7tv.app/emote/{id}/2x.webp" width="28" height="28" alt="{name}" style="vertical-align: middle; display: inline-block">` (WEBP only — no AVIF — RESEARCH.md Pitfall 4)
- FFZ emotes: `<img src="https://cdn.frankerfacez.com/emote/{id}/2" width="28" height="28" alt="{name}" style="vertical-align: middle; display: inline-block">`
- All emote `<img>` elements must have `alt` set to the emote code/text for accessibility. No `onerror` fallback needed — broken images show nothing rather than broken icon.

### Chat Toggle

- Trigger: keyCode 403 (Red button) anywhere in PlayerScreen (RESEARCH.md Pattern 7).
- Toggle state: `createSignal<boolean>(true)` — chat visible by default when stream starts playing.
- When toggled off: `<Show when={chatVisible()}>` — ChatSidebar is unmounted (not hidden). Video area expands to full width. WebSocket connection lives in TwitchChatService which is NOT inside ChatSidebar — it stays alive regardless of visibility.
- Toggle hint: A small label in the bottom-right corner of the video area (inside the safe zone): "Red — toggle chat". Visible for 3 seconds after stream starts, then fades out. Appears briefly again each time chat is toggled. 20px label size, --color-text-disabled, no background.

## States and Interactions

### Chat sidebar states

| State | Visual |
|-------|--------|
| Connecting | No sidebar rendered. WebSocket not yet subscribed. Video is full-width. |
| Loading emotes | Sidebar panel renders with "Loading chat..." text centered, --color-text-secondary, 20px. Video shrinks to accommodate sidebar. |
| Active (messages flowing) | Full sidebar with message list. Newest at bottom. Video area is `calc(100vw - 360px)` wide. |
| Connection error / reconnecting | Sidebar remains visible with existing messages. A small status line at top of panel: "Reconnecting..." in --color-text-secondary. No error screen — chat is secondary; playback continues. |
| Scope error (missing user:read:chat) | Full-screen error overlay (replaces PlayerScreen content, same visual treatment as existing error overlay): heading "Chat access required", body "Your login needs to be updated to show chat. Press OK to log out and sign in again.", single "Sign in again" button using --color-accent. |
| Chat hidden (toggled off) | Sidebar unmounted. Video expands to full width. Toggle hint appears briefly in video area. |
| No messages yet (channel just joined) | Sidebar renders. Message area empty. No empty-state message — blank is correct (chat is live, just quiet). |

### Info bar behavior

The info bar overlays the video area only (left portion of the screen). It does not extend into or interact with the chat sidebar. No conflict avoidance logic needed — they occupy separate layout regions.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (scope error) | "Sign in again" |
| Loading state | "Loading chat..." |
| Reconnecting status | "Reconnecting..." |
| Scope error heading | "Chat access required" |
| Scope error body | "Your login needs to be updated to show chat. Press OK to log out and sign in again." |
| Chat toggle hint | "Red — toggle chat" |
| Emote load failure (silent) | No visible copy — emote images simply do not render; their alt text is the code. |

Empty state: No copy needed. A blank chat area is the correct representation of a quiet channel. Adding "No messages yet" would be misleading (might still be connecting).

Destructive actions: None in Phase 5. The scope-error "Sign out" flow is handled by calling the existing `TwitchAuthService.clearTokens()` — no confirmation dialog required because the user explicitly chose it from the error state.

Source: copywriting patterns derived from existing PlayerScreen.tsx error overlay copy style (active verbs, problem + resolution path). REQUIREMENTS.md CHAT-01 through CHAT-04.
